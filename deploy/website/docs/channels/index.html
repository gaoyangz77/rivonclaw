聊天渠道

DlxAI 的核心是「一个 agent 出现在你所有的聊天 App 里」。下面是支持的渠道列表和接入方法。

支持的渠道

  • Telegram —— Bot Token 接入，几乎零门槛
  • Discord —— Bot Token + 服务器邀请
  • Slack —— Slack App 或 Bot Token
  • 飞书 / Lark —— App ID + App Secret
  • iMessage —— 仅 macOS，使用系统消息中继
  • LINE —— LINE Messaging API
  • Matrix —— Matrix 服务器 + access token
  • Mattermost —— 自托管 Mattermost 实例
  • 移动端（Mobile） —— 通过扫码配对你的手机
  • WeChat（个人微信） —— 扫码登录

添加一个渠道

  1. 左侧导航点「渠道」
  2. 顶部下拉框选你要接入的平台
  3. 点「连接」打开配置表单
  4. 填入对应的 token / app id 等凭证
  5. 保存 —— 立即生效，不需要重启

密钥怎么存的

所有 token 和 secret 都通过操作系统的安全存储加密：

  • macOS：Keychain
  • Windows：DPAPI（Data Protection API）
  • Linux：Secret Service / fallback 加密文件

本地 SQLite 数据库里只存非敏感配置，secret 永远不会以明文落盘。

多账号 / 多团队

每个渠道都支持多个账号。比如你可以同时接入两个 Slack workspace 或两个 Telegram bot， 每个账号有独立的 DM 策略和群组白名单。

群组安全： DM 默认是「pairing only」（仅响应已配对用户），群组默认是「allowlist only」（仅响应白名单群）。 这是为了避免陌生人无限调用你的 LLM 配额。
